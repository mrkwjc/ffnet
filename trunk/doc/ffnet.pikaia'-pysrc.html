 30      """ 
 31      Pikaia version 1.2 - genetic algorithm based optimizer. 
 32       
 33      Wrapped with f2py by Marek Wojciechowski 
 34      <mwojc@p.lodz.pl> 
 35       
 36      Original fortran code: Paul Charbonneau & Barry Knapp 
 37      <paulchar@hao.ucar.edu> 
 38      <knapp@hao.ucar.edu> 
 39       
 40      Simplest usage: 
 41          from pikaia import pikaia 
 42          x = pikaia(ff, n) 
 43       
 44      Function ff is a user-supplied scalar function of n vari- 
 45      ables, which must have the calling sequence f = ff(x, [n], [args]), 
 46      where x is a real parameter array of length n and [args] are 
 47      optional parameters. This function must 
 48      be written so as to bound all parameters to the interval [0,1]; 
 49      that is, the user must determine a priori bounds for the para- 
 50      meter space, and ff must use these bounds to perform the appro- 
 51      priate scalings to recover true parameter values in the 
 52      a priori ranges. 
 53       
 54      By convention, ff should return higher values for more optimal 
 55      parameter values (i.e., individuals which are more "fit"). 
 56      For example, in fitting a function through data points, ff 
 57      could return the inverse of chi**2. 
 58       
 59      Additional ff parameters [args] can be passed to pikaia routine 
 60      via keyword argument ff_extra_args which have to be a tuple 
 61      of variables. 
 62       
 63      Returning array x is the "fittest" (optimal) solution found, 
 64      i.e., the solution which maximizes fitness function ff. 
 65       
 66      Algorithm control parameters are set via keyword arguments: 
 67      individuals  - number of individuals in a population (default 
 68                     is 100) 
 69      generations  - number of generations over which solution is 
 70                     to evolve (default is 500) 
 71      digits       - number of significant digits (i.e., number of 
 72                     genes) retained in chromosomal encoding (default 
 73                     is 6)  (Note: This number is limited by the 
 74                     machine floating point precision.  Most 32-bit 
 75                     floating point representations have only 6 full 
 76                     digits of precision.  To achieve greater preci- 
 77                     sion this routine could be converted to double 
 78                     precision, but note that this would also require 
 79                     a double precision random number generator, which 
 80                     likely would not have more than 9 digits of 
 81                     precision if it used 4-byte integers internally.) 
 82      crossover    - crossover probability; must be  <= 1.0 (default 
 83                     is 0.85). If crossover takes place, either one 
 84                     or two splicing points are used, with equal 
 85                     probabilities 
 86      mutation     - 1/2/3/4/5 (default is 2) 
 87                     1=one-point mutation, fixed rate 
 88                     2=one-point, adjustable rate based on fitness 
 89                     3=one-point, adjustable rate based on distance 
 90                     4=one-point+creep, fixed rate 
 91                     5=one-point+creep, adjustable rate based on fitness 
 92                     6=one-point+creep, adjustable rate based on distance 
 93      initrate     - initial mutation rate; should be small (default 
 94                     is 0.005) (Note: the mutation rate is the proba- 
 95                     bility that any one gene locus will mutate in 
 96                     any one generation.) 
 97      minrate      - minimum mutation rate; must be >= 0.0 (default 
 98                     is 0.0005) 
 99      maxrate      - maximum mutation rate; must be <= 1.0 (default 
100                     is 0.25) 
101      fitnessdiff  - relative fitness differential; range from 0 
102                     (none) to 1 (maximum).  (default is 1.) 
103      reproduction - reproduction plan; 1/2/3=Full generational 
104                     replacement/Steady-state-replace-random/Steady- 
105                     state-replace-worst (default is 3) 
106      elitism      - elitism flag; 0/1=off/on (default is 0) 
107                     (Applies only to reproduction plans 1 and 2) 
108      verbosity    - printed output 0/1/2=None/Minimal/Verbose 
109                     (default is 0) 
110       
111      Thus more complex usage: 
112          x = pikaia( ff, n, ff_extra_args = (1, 0.5), \  
113                      individuals = 200, \  
114                      generations = 50, \  
115                      verbosity = 1 ) 
116                       
117      """ 
118

122      del randint

124      # Restore control array

127   
128      # Optimize 
129      x, f, status = _pikaia.pikaia(ff, n, ctrl, ff_extra_args) 
130      return x
